# Drive Access Console v1.0

## Release Summary
Drive Access Console v1.0 establishes the first production-ready baseline of the RBAC control plane for Google Shared Drives at Conceivable. This release delivers policy modeling, role-driven access assignment, restricted-folder controls, Google Workspace integration, monitoring, reconcile workflows, reporting, and operational cleanup tools in one end-to-end system.

## Included In v1.0

### Core RBAC administration
- App role assignment for administrative and audit users.
- Business access role assignment for company-facing access models.
- Role-to-drive policy mapping for both app roles and business roles.
- Editable role matrix for non-restricted Shared Drive access using `Viewer` and `Contributor` mappings.

### Restricted access controls
- Restricted-folder policy modeling in the RBAC catalog.
- Automation for granting restricted folder access through role mappings.
- Exception-based restricted access request flow with approval and audit trail.
- Current QMS restricted folder baseline aligned to:
  - `01_QualityAssurance_Working / 05_Audits`

### Google Workspace / Google Drive integration
- Real Google login flow on custom domain.
- Google Group membership add/remove automation.
- Managed Google Groups member visibility in the portal.
- Shared Drive and restricted folder ACL monitoring from Google Drive.
- Drift detection between RBAC policy and actual Google ACL state.
- Reconcile preview and apply flow for supported drift corrections.

### User lifecycle operations
- Active employee sync from Google Directory into `grp-all-employees`.
- Offboarding hygiene workflow for stale RBAC-managed access.
- Detection and cleanup of orphaned group memberships without local assignments.

### Reporting and auditability
- Audit log-backed change tracking across assignments, access requests, reconcile, sync, and cleanup actions.
- `Access & Reconcile Change Log` report improvements.
- Quarterly Access Review generation.
- Quarterly reports archived into quarter folders such as:
  - `2026_Q1_Reports`
- Safe automation entrypoint for quarterly report generation.

### Templates and operational tooling
- Operational folder template creation from the portal.
- Engineering project template expanded with subsystem substructure under `33_Subsystems`.

### UX and platform improvements
- Minimal visual refresh across the console.
- Fixed sidebar with independent workspace scrolling on desktop.
- Clearer separation between policy configuration, local role assignments, and active derived memberships.
- Improved inline feedback for reconcile and orphaned membership cleanup actions.

## Operational Baseline For v1.0
- Shared Drive monitor aligned and usable from the portal.
- Restricted-folder policy and ACL monitoring aligned for:
  - `01_QualityAssurance_Working / 05_Audits`
  - `04_Support_Working / 01_HumanResources`
  - `04_Support_Working / 05_Finance`
  - `04_Support_Working / 06_Legal`
- Custom-domain authentication operational at `rbac.conceivable.life`.

## Known Constraints
- App role and business role matrix editing is currently limited to non-restricted Shared Drive mappings.
- Restricted-folder matrix editing is still controlled outside the inline editor flow.
- Quarterly access review content depends on populated review items in the active quarter dataset.
- Some operational workflows still benefit from explicit preview and verification before broad rollout.

## Recommended Next Steps After v1.0
1. Add bulk cleanup for orphaned memberships.
2. Add impact preview for role-matrix changes before applying policy changes broadly.
3. Expand editable matrix support for restricted-folder mappings with stronger safeguards.
4. Continue hardening scheduled operations for directory sync, offboarding, and quarterly reporting.
